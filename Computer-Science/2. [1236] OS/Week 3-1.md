# 3. 운영체제의 내부 구조
## 1. MS-DOS
- 모든 기능을 Kernal에 넣음 => 취약
## Layered Approach
- 기능을 Layer로 나눔
- 바로 아래 Layer의 기능만 사용 가능
- 디버깅 쉬움
- layer 나누기 불가 / 속도느림, 메모리사용량 많음
## Microkernel
- Kernel의 기능들을 User space로 뺌
- 확장이 쉽고 안전함
- 하지만 Message passing 사용 => Overhead(성능저하)
## Module
- 현대 운영체제
- 각 핵심 요소들이 Module로 이루어져 넣고 빼는 것이 자유로움
- 메시지 패싱도 X, 동적 할당 => 유연하고 성능도 좋음
