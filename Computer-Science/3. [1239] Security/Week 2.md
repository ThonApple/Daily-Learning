# 운영체제 2주차

## 1. 정보보호의 핵심 3가지
### 1. 기밀성 (Confidentiality)
- 비인가 사용자의 `읽기`권한 제한
### 2. 무결성 (Integrity)
- 비인가 사용자의 `쓰기`권한 제한
### 3. 가용성 (Availability)
- 소비자가 원할때 정보를 제공
### 인증 및 인가
- 인증 : 사용자가 누구인지 확인
- 인가 : 권한 부여
### ETC. DDOS
- DOS : 자원을 쓸모 없는 곳에 소모시킴 -> 서비스 다운 공격
- DDOS : (Distributed) 여러개 분산

## 2. 암호학 (Crypto)
### 암호화 Cryptography / 암호해독 Cryptanalysis
- 원본(Plaintext), 암호문(Ciphertext), 키(Key)
- 대칭키 암호 : 암호화 키와 복호화 키가 동일함
- 공개키 암호 : 암호화 키와 복호화 키가 다름 (공개키, 비밀키)
- 케르크호프스의 원리 : 알고리즘은 공개되어도 안전해야 하며, 오직 키만 비밀이어야 함. 알고리즘을 숨기는 것은 오히려 취약점 발견을 늦춰 위험할 수 있음

## 3. 고전 암호의 종류와 특징
### 치환 암호 (Substitution Cipher)
- 글자를 다른 글자로 바꾸는 방식
- `카이사르 암호 (Caesar's Cipher)` : 고정된 값(보통 3칸)만큼 알파벳을 미는 방식 (최초의 암호 알고리즘)
- `단순 치환 암호 (Simple Substitution)` : 키 공간이 $26!$로 매우 커서 전수조사로는 해독이 불가능함
### 전치 암호 (Transposition Cipher)
- Key : `사이즈`와 `어떻게 바꿨는지` row(3,5,1,4,2) and col(1,3,2)
- 글자를 바꾸지 않고 위치만 이동시키는 방식
- 특징: 글자의 빈도수는 유지되지만 위치가 섞임 (이름만 바뀜) 
### 일회용 패드 (One-time Pad)
- 랜덤한 키를 사용하여 평문과 XOR 연산을 수행.
- Fake Key를 주면 틀린 답이 나와 해독하지 못함.
- 이는 이론적으로 완벽하게 안전하지만, 키를 매번 안전하게 공유하기 어렵고 재사용이 불가능해 실용성이 떨어짐
- 같은 키가 여러번 쓰이면 해독(간파)당함.
