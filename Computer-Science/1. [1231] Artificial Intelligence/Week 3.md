# 인공지능 3주차

# 1. 에이전트의 유형 4가지
### 고려사항
1. 효율성 (노력)
2. 반응성 (속도)
3. 자율성 (스스로)
4. 유연성 (변수대처)

## 1. 단순 반응형 에이전트 
1. Sensors : 센서 인식 (상태 추정)
2. Rules : 조건-행동 규칙
3. Effectors : 행동 결정
### 장점
- 쉽고 빠름 (효율성, 반응성 높음)
### 단점
- 불학실환 센싱(과거, 미래 고려하지 않음)
- Rule을 통해 행동을 미리 제공해야 함 (자율성, 유연성 없음)

## 2. 모델-기반 반응형 에이전트
1. `State` : 환경 상태 정보 유지
### 장점
1. 과거, 변화 고려 (정확한 상태파악)
2. 빠른 행동결정 (효율성, 반응성)
### 단점
1. 여전히 자율성과 유연성은 없음.

## 3. 목표-기반 에이전트
1. Sensors : 센서 인식 (상태 추정)
2. State : 환경 상태 정보 유지
3. `Goals` : 목표만 존재 (규칙X)
4. Effectors : 행동 결정

### 장점
1. 정확한 환경파악, 행동 결과 예측 후 결정(정확)
2. 목표 달성을 위한 효과적 행동
3. 탐색, 자동계획을 통해 스스로 결정 (자율성, 유연성 높음)
### 단점
1. 복잡하고 느림(효율성, 반응성 낮음)

## 4. 유틸리티-기반 에이전트
1. Sensors : 센서 인식 (상태 추정)
2. State : 환경 상태 정보 유지
3. `Utility` : 유용성(가치)
4. Effectors : 행동 결정

### 장점
1. 유틸리티(이득, 가치) 평가 => 높은 가치를 위한 행동
2. 합리적 (자율성, 유연성)
### 단점
1. 동적이고 복잡한 환경 => 유틸리티 평가 난이 => (효율성, 반응성 낮음)

# 2. 문제 해결 탐색 기술
## 1. 문제 해결 에이전트
### 1-1. 목표와 문제 인식
- `Goal`은 외부에서 주어지지만 `Sub-Goal`은 에이전트 스스로 세움.
- '문제'란 `현재 상태`와 `목표 상태`의 간격을 알아내고, 좁히는 것.
### 1-2. 탐색(Search)
- 행동 지식을 바탕으로 시뮬레이션
- 목표에 도달하는 `행동의 순서를 찾아내는 과정`
### 1-3. 실행과 수정
- 계획 수립 후에는 첫번째 행동만 반복하게 됨. 
- 현실 환경은 Dynamic하기 때문에 계획을 `지속적으로 수정해야 함`

# 2. 상태 공간 탐색 문제
- 현실 문제(자연어) => '알고리즘에 어떻게 적용, 응용할지'가 중요
### State (상태, 모든 경우의 수)
- 보이는 것(Percept)와 다른 환경 정체를 아우르는 정보
- ex) 로봇의 위치(2) * A방의 청결도(2) * B방의 청결도(2) = 8
### Action (변화시키는 조작)
- 상태를 변화시키는 조작
- 숫자 타일 8개를 각각 움직인다고 하면 `9!`라는 어마무시한 경우의 수가 생김
- `빈칸 하나`가 상하좌우로 움직이는 경우 => `가짓수 4개`로 단순화
### Initial State(초기상태), Goal condition(목표), Path cost(비용,시간), Solution(정답)

# 3. 트리 탐색
- 초기 상태 : Root
- 할 수 있는 Action을 적용해 미래의 상태들을 계속 Expand하며 Goal에 도달
  ### Node
  - 단순한 State가 아님. `부모 노드`, `도달 비용`, `적용 행동` 등의 정보들을 담고있음.

# 4. 탐색 전략 알고리즘
- 어떤 노드에 먼저 갈까?(전략)
### 1. 무정보 탐색 (AI는 사용하지 않음)
- 목표에 대한 정보 없이 순서대로, 맹목적으로 다 뒤짐
- 너비 우선 탐색 BFS / 깊이 우선 탐색 DFS / 균일 비용 탐색 USC / 반복적 깊이 증가 탐색 IDS
### 2. 정보 / 휴리스틱(경험적) 탐색
- 어느 쪽으로 가야 목표에 가까운지 경험적 지식(Heuristic)에 의거해 힌트를 얻고 유망한 쪽을 탐색
- AI는 방대한 상태, 공간을 다루기 때문에 무조건 이 방식으로 `효과적으로` 탐색 범위를 줄여야 함.
